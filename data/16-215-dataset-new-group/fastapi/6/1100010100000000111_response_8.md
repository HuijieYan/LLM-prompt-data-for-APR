## Analysis:
The GitHub issue describes a problem where repeated key=value pairs in form data are not being handled correctly by FastAPI's `Form` validation. The issue suggests that FastAPI should collect repeated keys in the 2-tuple list that `request.form()` gives and assign those values as a list to the same key before validation happens.

## Potential error locations:
1. Handling of repeated key=value pairs in form data.
2. Validaion of the form data against the field schema.

## Cause of the bug:
The `request_body_to_args` function does not handle repeated key=value pairs in form data correctly. It only captures the last key=value pair, instead of collecting all values for the same key and assigning them as a list before validation.

## Strategy for fixing the bug:
To fix the bug, we need to modify the `request_body_to_args` function to correctly handle repeated key=value pairs in the form data. We should ensure that all values for the same key are collected and assigned as a list before validation against the field schema.

## Corrected version of the function:
```python
async def request_body_to_args(
    required_params: List[ModelField],
    received_body: Optional[Union[Dict[str, Any], FormData]],
) -> Tuple[Dict[str, Any], List[ErrorWrapper]]:
    values = {}
    errors = []
    if required_params:
        for field in required_params:
            value: Any = None
            if received_body is not None:
                if field.shape in sequence_shapes and isinstance(
                    received_body, FormData
                ):
                    values[field.name] = received_body.getlist(field.alias)
                else:
                    value = received_body.get(field.alias)
                    if value is not None:
                        values[field.name] = value
            if field.required and (value is None or value == ""):
                if PYDANTIC_1:
                    errors.append(
                        ErrorWrapper(MissingError(), loc=("body", field.alias))
                    )
                else:  # pragma: nocover
                    errors.append(
                        ErrorWrapper(  # type: ignore
                            MissingError(),
                            loc=("body", field.alias),
                            config=BaseConfig,
                        )
                    )
            else:
                if field_info in sequence_shapes:
                    v_, errors_ = field.validate(values[field.name], values, loc=("body", field.alias))
                else:
                    v_, errors_ = field.validate(value, values, loc=("body", field.alias))
                if isinstance(errors_, ErrorWrapper):
                    errors.append(errors_)
                elif isinstance(errors_, list):
                    errors.extend(errors_)
                else:
                    values[field.name] = v_
    return values, errors
```

In the corrected version, we check if the form data `received_body` is of type `FormData` and collect all values for the same key as a list using `received_body.getlist(field.alias)`. Then we perform validation against the field schema and accumulate any errors.

This corrected version should handle repeated key=value pairs in form data correctly, addressing the issue raised in the GitHub post.
