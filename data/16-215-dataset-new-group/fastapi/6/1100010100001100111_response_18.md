## Bug Explanation
The bug in the `request_body_to_args` function is related to handling repeated keys in form data. The function doesn't properly handle multiple values for the same key in the `FormData` object. As a result, when there are repeated keys in the form data, only the last key=value pair is considered, leading to incorrect results. This issue is in line with the reported GitHub issue, which requests support for repeated key=value pairs in form data.

## Bug Fix Strategy
To fix the bug, we need to modify the `request_body_to_args` function to correctly handle repeated keys in form data. Specifically, the function should aggregate all the values for a repeated key into a list and assign that list to the key before validation.

## Corrected Version of the Function
```python
import asyncio
from pydantic.error_wrappers import ErrorWrapper
from pydantic.errors import MissingError
from pydantic.fields import Field as ModelField
from starlette.datastructures import FormData
from typing import Any, Dict, List, Tuple, Union

async def request_body_to_args(
    required_params: List[ModelField],
    received_body: Optional[Union[Dict[str, Any], FormData]],
) -> Tuple[Dict[str, Any], List[ErrorWrapper]]:
    values = {}
    errors = []
    if required_params and received_body:
        for field in required_params:
            if field.alias in received_body:
                value = received_body.getlist(field.alias) if isinstance(received_body, FormData) else [received_body[field.alias]]
                if field.required and not value:
                    errors.append(ErrorWrapper(MissingError(), loc=("body", field.alias)))
                else:
                    v_, errors_ = field.validate(value, values, loc=("body", field.alias))
                    if isinstance(errors_, ErrorWrapper):
                        errors.append(errors_)
                    elif isinstance(errors_, list):
                        errors.extend(errors_)
                    else:
                        values[field.name] = v_
            else:
                if field.required:
                    errors.append(ErrorWrapper(MissingError(), loc=("body", field.alias)))
                else:
                    values[field.name] = deepcopy(field.default)
    return values, errors
```

In the corrected version of the function, we iterate through the required parameters and check if each field's alias exists in the `received_body`. If the alias exists, we use `received_body.getlist` to get all the values for that alias as a list. We then proceed to validate and process the values as needed. This modification ensures that all values for repeated keys are correctly handled and aggregated into lists before validation.
