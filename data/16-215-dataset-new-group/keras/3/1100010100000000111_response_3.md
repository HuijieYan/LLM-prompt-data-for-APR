### Analysis:
The bug in the `clone_model()` function results in an "AssertionError" with the message "Could not compute output Tensor" when using `multi_gpu_model` with `cpu_relocation=True`. This error occurs due to the `output_masks` being always [None], but [None, None] is expected, which is caused by a layer not supporting masks.

### Potential error locations:
1. Line 157: Loop to zip `reference_output_tensors` and `output_tensors` to generate `output_masks`.
2. Line 153: Calculating `output_masks` using `layer.compute_mask`, which results in the incorrect behavior.

### Cause of the bug:
The bug is due to the `layer.compute_mask` always returning None, leading to the 'Could not compute output Tensor' error. This occurs when using a functional model with a layer that has multiple outputs without mask support.

### Strategy for fixing the bug:
To fix the bug, we need to handle the case where a layer does not support masks, adjust the behavior of computing the `output_masks`, and ensure the model cloning process handles the absence of masks gracefully.
