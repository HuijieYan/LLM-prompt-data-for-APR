Fixing the bug: 
The function quantile returns unexpected values when working with dates.

The cause of the bug:
The function `_get_numeric_data` is called to filter out only numeric columns. For datetime columns, it calls `_data.quantile`, and if the data has been transposed, the function tries to reshape it to Series or DataFrame. This causes the problem when it tries to concatenate the data and throws a ValueError.

Fixing the bug:
To solve the bug, it would be best to not reshape the data when it is transposed during the calculation and reshape it only in the end before returning.

Below is a corrected version of the function:

```python
def quantile(self, q=0.5, axis=0, numeric_only=True, interpolation="linear"):
    self._check_percentile(q)
    
    data = self._get_numeric_data() if numeric_only else self
    axis = self._get_axis_number(axis)
    
    result = data._data.quantile(q, axis, interpolation)
    
    if result.ndim == 2:
        result = self._constructor(result, index=self.columns)
    else:
        result = self._constructor_sliced(result, index=self.columns, name=q)
        
    return result
```
