The bug is demonstrated in the GitHub issue where the `pd.DataFrame(pd.date_range('1/1/18', periods=5)).quantile()` call raises a `ValueError` instead of returning the quantile value similar to how it would work for a Series with datetime data.

The bug is caused by the fact that the `_check_percentile` method is not checking for datetime or timedelta type of data before the calculation, and thus it tries to perform quantile calculations on non-numeric data, resulting in the `ValueError`.

To resolve this issue, the `_check_percentile` method should be updated to check for non-numeric data and handle datetime and timedelta data differently before performing the quantile calculations.

```python
        data = self

    is_transposed = axis == 1

    if is_transposed:
        data = data.T

    # Check for datetime and timedelta data

        # Get only the numeric columns

    result = data._data.quantile(
        qs=q, axis=1, interpolation=interpolation, transposed=is_transposed
    )

        result = self._constructor(result)
    else:

    if is_transposed:
        result = result.T

    return result
```
