### Identifying the bug
The buggy function `info` is attempting to retrieve the version of the Fish shell using the `Popen` method, but it's not handling the `stdout` properly, which causes the function to return an incorrect value. This bug is causing a recursive loop when used with Oh-My-Fish's TheFuck plugin, leading to hanging of the Fish Shell initialization.

### Bug cause
The `Popen` method is used to execute the command `fish -c 'echo $FISH_VERSION'` and capture its output. However, the code doesn't properly handle the `stdout` and its data retrieval, leading to incorrect behavior.

### Strategy for fixing the bug
To fix the bug, we need to properly handle the `stdout` data retrieval from the `Popen` method and ensure that the version is extracted correctly. Additionally, the code needs to be modified to address the recursive loop issue when used with Oh-My-Fish's TheFuck plugin.

from ..utils import DEVNULL, cache

class Fish(Generic):
