The buggy function `convert_image` is intended to convert the input image to RGB mode and resize it if a specific size is provided. However, it has a bug where it incorrectly checks for the image format and mode, and it does not handle the RGBA mode correctly. 

## Bug Analysis
1. The first if condition checks for `image.format == 'PNG'` instead of `'JPEG'`, which is incorrect.
2. The second if condition converts the image to RGB mode if it's not already in RGB mode. This is problematic because it will also convert images in RGBA mode to RGB, which is not the intended behavior.
3. The addition of `background.paste(image, image)` is incorrect and unnecessary. It should be `background.paste(image, (0,0), image)`.

## Bug Fix Strategy
1. Change the condition from `image.format == 'PNG'` to `image.format == 'JPEG`.
2. Use a separate condition to handle the RGBA mode specifically and convert it to RGB mode without changing the other non-RGB modes.

```python
def convert_image(self, image, size=None):
    if image.format == 'JPEG' and image.mode == 'RGBA':
        background = Image.new('RGB', image.size, (255, 255, 255))
        background.paste(image, (0, 0), image)
        image = background

    if image.mode != 'RGB':
        image = image.convert('RGB')

    if size:
        image = image.copy()
        image.thumbnail(size, Image.ANTIALIAS)

    buf = BytesIO()
    image.save(buf, 'JPEG')
    return image, buf
```
