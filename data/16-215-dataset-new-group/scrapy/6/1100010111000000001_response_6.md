The bug in the convert_image function seems to be related to the conditional logic for handling PNG images with RGBA mode. When the condition is met, the function creates a new RGBA background and pastes the original image onto it, then converts it to RGB. However, the paste function's second argument should be the mask, not the source image.

The strategy for fixing the bug would be to modify the background.paste(image, image) line to background.paste(image, (0, 0), image) to correctly paste the image onto the background.

Here's the corrected version of the convert_image function:

```python
def convert_image(self, image, size=None):
    if image.format == 'PNG' and image.mode == 'RGBA':
        background = Image.new('RGBA', image.size, (255, 255, 255))
        background.paste(image, (0, 0), image)
        image = background.convert('RGB')
    elif image.mode != 'RGB':
        image = image.convert('RGB')

    if size:
        image = image.copy()
        image.thumbnail(size, Image.ANTIALIAS)

    buf = BytesIO()
    image.save(buf, 'JPEG')
    return image, buf
```
