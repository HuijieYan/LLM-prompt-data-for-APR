The bug in the `convert_image` function is that it converts the image to RGB format without considering the existing alpha channel, resulting in a loss of transparency for PNG images with RGBA mode.

To fix the bug, we need to check if the input image is in PNG format and has an RGBA mode. If so, we need to create a new RGBA image with a white background, paste the input image onto the new background, and then convert the result to an RGB mode image. If the input image is not in PNG format or does not have an RGBA mode, we should simply convert it to an RGB mode image.

Here's the corrected version of the `convert_image` function:

```python
def convert_image(self, image, size=None):
    if image.format == 'PNG' and image.mode == 'RGBA':
        background = Image.new('RGBA', image.size, (255, 255, 255, 255))
        background.paste(image, (0, 0), image)
        image = background.convert('RGB')
    else:
        image = image.convert('RGB')

    if size:
        image = image.copy()
        image.thumbnail(size, Image.ANTIALIAS)

    buf = BytesIO()
    image.save(buf, 'JPEG')
    return image, buf
```
