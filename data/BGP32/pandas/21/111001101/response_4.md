### Bug Explanation
The bug occurs in the `_get_with` function when the `key` parameter is not recognized as a list-like object due to invalid handling of scalar values. This leads to an incorrect inference of the `key_type`, causing the function to return the wrong result.

### Bug Fix Strategy
To fix the bug, we need to ensure that scalar values are properly recognized as list-like objects before inferring the `key_type`. This can be achieved by adding a check to handle scalar values correctly.

    elif isinstance(key, tuple):

    elif not is_list_like(key) and not isinstance(key, str):
