1. The `level` parameter passed to the `_unstack_multiple` function is expected to be a string name of the level. However, the function tries to find the index of the level name within the `names` attribute of the MultiIndex directly, which causes a ValueError.
2. The condition for checking if the `level` is an integer is misplaced and should be checked before trying to find the level number.
3. The error handling logic for a level not being found is throwing a KeyError instead of a ValueError.
