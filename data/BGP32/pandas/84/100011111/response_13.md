### Bug Analysis:
The bug occurs because the function `_get_level_number()` is unable to handle tuple names when looking up a level in the MultiIndex. This leads to a KeyError being raised with the message 'Level [level_name] not found'.

### Issues in the Buggy Function:
1. The function does not handle well the case when the names of the levels in the MultiIndex are tuples.
2. Transformation of names to numbers for levels using `_get_level_number` is done inappropriately.
3. Handling of multiple levels in the presence of MultiIndex columns is not accurate.

### Bug Fix Strategy:
1. Adjust the code to correctly handle tuple names in the MultiIndex levels.
2. Modify the way levels are translated into numbers using `_get_level_number`.
