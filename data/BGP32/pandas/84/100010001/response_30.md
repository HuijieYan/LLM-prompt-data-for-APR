### Analysis and Bug Explanation
The bug occurs in the `_get_level_number` method when trying to find the index of a level that contains a tuple in the `names` attribute of the MultiIndex. The bug arises because the `level` parameter passed to `_get_level_number` is a tuple instead of a single level name. This causes the error when the method tries to find a tuple in the list of level names.

### Bug Location
The bug is located in the `_get_level_number` method where `self.names.index(level)` is expecting a single string as the `level`, not a tuple.

### Bug Fix Strategy
To fix the bug, we need to handle the case where the `level` parameter passed to `_get_level_number` is a tuple. We should check if the `level` is a tuple, extract the appropriate name from the tuple, and then find its index in the `names` list.
