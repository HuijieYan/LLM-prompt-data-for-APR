## Analysis:
- The error occurs in the `_get_level_number` method when trying to retrieve the index of a specified level from the names of a MultiIndex object.
- The error messages vary between `ValueError: 'A' is not in list` and `KeyError: 'Level A not found`.

## Bug:
- The method `_get_level_number` expects a level name as input but mistakenly treats the input as an index.
- The code doesn't handle the case where the level name is a tuple within the names list of the MultiIndex object.

## Fix:
1. Update the `_get_level_number` function to handle the case where the input level corresponds to a tuple within the names list of the MultiIndex object.
2. Extract the name component of the tuple before proceeding to find the index.
