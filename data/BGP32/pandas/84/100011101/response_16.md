### Bug Analysis
The bug arises from the fact that the function `_get_level_number` is expecting a single level name but is receiving a tuple of level names. This leads to errors in the process of unstacking the data when dealing with multi-level index columns.

### Bug Fix Strategy
To fix the bug, we need to modify the section of the code where the function tries to retrieve the level number for each column. Instead of passing a tuple `(level1, level2, ...)` for column names, we need to pass individual level names to the `_get_level_number` function.

def _unstack_multiple(data, clocs, fill_value=None):

        return data

    # NOTE: This doesn't deal with hierarchical columns yet

    index = data.index

    clocs = [index._get_level_number(i) for i in clocs]

    if isinstance(clocs[0], tuple):
        clocs = [index._get_level_number(i) for i in clocs[0]]

    cnames = [index.names[i] for i in clocs]

    rcodes = [index.codes[i] for i in rlocs]
    rnames = [index.names[i] for i in rlocs]

        )

    if isinstance(data, Series):

        dummy.index = dummy_index

    if isinstance(unstacked, Series):
        unstacked.index = new_columns
