## Bug Analysis
The bug occurs in the `_clone_functional_model` function when trying to clone a model that contains multiple output tensors. The function fails to properly handle the case where a layer generates more than one output, leading to an assertion error stating that it could not compute the output.

### Issues in the Buggy Function
1. The function does not support multiple output tensors from a single layer.
2. Handling the creation of input layers and mapping input tensors is not done correctly.
3. The tensor map does not account for multiple output tensors generated by a layer.

### Bug Cause
The `SwapLayer` instance defined in the test function produces two output tensors when called, leading to two separate output tensors from a single layer. The function fails to correctly map these output tensors, resulting in the assertion error when trying to compute the outputs of the model.

## Bug Fix Strategy
1. Modify the function to handle cases where a layer generates multiple output tensors.
2. Ensure that the mapping of input and output tensors is correct.
3. Update the tensor map to handle multiple output tensors generated by a layer.

## Corrected Function
```python
def _clone_functional_model(model, input_tensors=None):
    if not isinstance(model, Model):
        raise ValueError('Expected `model` argument to be a `Model` instance, got ', model)
    if isinstance(model, Sequential):
        raise ValueError('Expected `model` argument to be a functional `Model` instance, got a `Sequential` instance instead:', model)

    layer_map = {}  # Cache for created layers
    tensor_map = {}  # Map {reference_tensor: (corresponding_tensor, mask)}

    if input_tensors is None:
        input_tensors = [Input(batch_shape=layer.input_shape) for layer in model.layers]
    else:
        input_tensors = to_list(input_tensors)
    
    for orig_layer, input_tensor in zip(model.layers, input_tensors):
        layer_map[orig_layer] = input_tensor

    for current_layer in model.layers:
        if current_layer not in layer_map:
            layer_map[current_layer] = current_layer.__class__.from_config(current_layer.get_config())
        
        for node in current_layer._inbound_nodes:
            reference_input_tensors = node.input_tensors
            computed_tensors = []
            for tensor in reference_input_tensors:
                if tensor in tensor_map:
                    computed_tensors.append(tensor_map[tensor][0])
            if len(computed_tensors) == len(reference_input_tensors):
                output_tensors = to_list(current_layer(computed_tensors))
                for x, y in zip(node.output_tensors, output_tensors):
                    tensor_map[x] = (y, None)

    output_tensors = [tensor_map[out][0] for out in model.outputs]
    return Model(input_tensors, output_tensors, name=model.name)
``` 

This corrected version of the function handles cases where a layer generates multiple output tensors, appropriately mapping input and output tensors. This should resolve the issue encountered in the failing test.
