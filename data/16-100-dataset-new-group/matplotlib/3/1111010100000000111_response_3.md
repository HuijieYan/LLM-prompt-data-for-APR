## Analysis and Identification of Errors

The buggy function `_recache()` is part of the `MarkerStyle` class and is responsible for re-calculating the marker style. The issue on GitHub indicates that when setting the `fillstyle` to 'none' in the `MarkerStyle`, the markers are not hollow as expected when using `ax.scatter()`.

The cause of the bug appears to be related to the `_recache()` function not handling the `fillstyle` parameter correctly. The function always sets `_filled` to `True`, regardless of the `fillstyle` parameter.

## Bug Cause
The bug is caused by the `_recache()` function not updating the `_filled` attribute based on the `fillstyle` parameter.

## Bug Fix Strategy
To fix the bug, the `_recache()` function should be modified to correctly handle the `fillstyle` parameter and update the `_filled` attribute accordingly.

## Corrected Version
```python
class MarkerStyle():
    def _recache(self):
        if self._marker_function is None:
            return
        self._path = _empty_path
        self._transform = IdentityTransform()
        self._alt_path = None
        self._alt_transform = None
        self._snap_threshold = None
        self._joinstyle = 'round'
        self._capstyle = 'butt'

        self._marker_function()
```
