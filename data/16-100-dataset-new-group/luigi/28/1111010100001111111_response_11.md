### Cause of the bug

The bug occurs because the `table_exists` function does not perform a case-insensitive check when comparing the table name obtained from `show tables` in Hive with the input table name. This leads to inconsistencies when the input table name and the table name in Hive have different cases. The GitHub issue indicates that the function should perform a case-insensitive check.

### Strategy for fixing the bug

To fix the bug, the `table_exists` function needs to compare the table names in a case-insensitive manner. This can be achieved by converting both the input table name and the table name obtained from Hive to lowercase before performing the comparison.

### Corrected version of the function

```python
class HiveCommandClient(HiveClient):
    """
    Uses `hive` invocations to find information.
    """

    def table_exists(self, table, database='default', partition=None):
        if partition is None:
            stdout = run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table))

            return stdout and table.lower() in stdout.lower()
        else:
            stdout = run_hive_cmd("""use %s; show partitions %s partition
                            (%s)""" % (database, table, self.partition_spec(partition)))

            if stdout:
                return True
            else:
                return False
```

In the corrected version, the table name obtained from Hive (`stdout`) and the input table name (`table`) are converted to lowercase using the `lower()` method before comparison. This ensures a case-insensitive check for table existence.
