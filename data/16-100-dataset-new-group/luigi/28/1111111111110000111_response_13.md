### Bug Fixing Strategy
The bug occurs because the table existence check is case sensitive, leading to test failures for tables with different case variations. The fix should involve making the table existence check case insensitive by comparing the table name and the output from Hive in a case-insensitive manner.


### The corrected version of the buggy function
```python
class HiveCommandClient(HiveClient):
    """
    Uses `hive` invocations to find information.
    """

    def table_exists(self, table, database='default', partition=None):
        if partition is None:
            stdout = run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table))

            return stdout and table.lower() in stdout.lower()
        else:
            stdout = run_hive_cmd("""use %s; show partitions %s partition
                                (%s)""" % (database, table, self.partition_spec(partition)))

            return bool(stdout)
```
