## Bugs in the code
1. The `table_exists` function doesn't convert the table name to lowercase before comparing with the output from `run_hive_cmd`.
2. There is a redundant `self.partition_spec` function call.

## Explanation
The failing test case `test_table_exists` fails because the second assertion `self.assertTrue(returned)` fails, which indicates that the table name comparison is not case-insensitive as expected.

The failing test case `test_apacheclient_table_exists` fails for the same reason.

## Bugs fixing strategy
1. Convert the `table` parameter to lowercase before comparison in the `table_exists` function.
2. Remove the redundant `self.partition_spec` function call.

## The corrected version of the function

```python
def table_exists(self, table, database='default', partition=None):
    if partition is None:
        stdout = run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table.lower()))
        return stdout and table.lower() in stdout.lower()
    else:
        stdout = run_hive_cmd("""use %s; show partitions %s partition
                            (%s)""" % (database, table.lower(), self.partition_spec(partition)))
        if stdout:
            return True
        else:
            return False
```
