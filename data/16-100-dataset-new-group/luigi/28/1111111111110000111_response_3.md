```python
import luigi.contrib.hive as hive

class HiveCommandClient(hive.HiveClient):

    def table_exists(self, table, database='default', partition=None):
        if partition is None:
            stdout = hive.run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table))

            return stdout and table.lower() in stdout.lower()
        else:
            stdout = hive.run_hive_cmd("""use %s; show partitions %s partition
                            (%s)""" % (database, table, self.partition_spec(partition)))

            if stdout:
                return True
            else:
                return False
```

By using the `table.lower()` to compare against `stdout.lower()`, we will have a case-insensitive comparison and the bug should be resolved. Adding this case-insensitive comparison addresses the issue reported in GitHub and allows the `table_exists` function to function as expected.
