The bug in the `table_exists` function is that it checks for the presence of the table name as a whole in the output of the `run_hive_cmd` function, instead of checking for each individual table name.

The bug can be fixed by splitting the stdout string obtained from `run_hive_cmd` by newline and then checking if the table name is in the list of obtained table names.

Here is the corrected version of the `table_exists` function:

```python
def table_exists(self, table, database='default', partition=None):
    if partition is None:
        stdout = run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table))

        return stdout and table in stdout.split('\n')
    else:
        stdout = run_hive_cmd(f"use {database}; show partitions {table} partition ({self.partition_spec(partition)})")

        return table in stdout.split('\n')
```
