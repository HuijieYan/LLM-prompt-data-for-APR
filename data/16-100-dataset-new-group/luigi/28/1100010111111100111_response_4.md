The function does not handle case insensitivity correctly, leading to failing test cases. The issue on GitHub suggests that the function should check `stdout` against the lowercase version of the table name.

### Bug Cause
The bug is caused by the function not handling case sensitivity correctly, leading to incorrect comparisons and failing test cases.

### Suggested Fix
Modify the function to convert both the `stdout` and the table name to lowercase before comparing them to make the comparison case insensitive.

def table_exists(self, table, database='default', partition=None):
    if partition is None:
        stdout = run_hive_cmd('use {0}; show tables like "{1}";'.format(database, table))

        return stdout and table.lower() in stdout.lower()
    else:
        stdout = run_hive_cmd("""use %s; show partitions %s partition
                            (%s)""" % (database, table, self.partition_spec(partition)))

        if stdout:
            return True
        else:
            return False
