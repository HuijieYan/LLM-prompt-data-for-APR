### Analysis

### Bug
The bug is in the logic of the `initialize` function. When `make_current` is `True`, it is wrongly checking if the current `IOLoop` instance is `None` instead of checking if it already exists.

- Additionally, if `make_current` is `None`, it should also check if the current `IOLoop` instance already exists before calling `self.make_current()`.
