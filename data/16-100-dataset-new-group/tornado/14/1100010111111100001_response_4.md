### Bug Explanation
The buggy function `initialize` in the `tornado/ioloop.py` file is supposed to initialize the `IOLoop` instance. The problem lies in the `initialize` function not correctly handling the `make_current` parameter. If `make_current` is `True`, it should check if an `IOLoop` instance already exists and raise a `RuntimeError` if it does.

### Bug Fix
To fix the bug, the `initialize` function should be modified to properly handle the `make_current` parameter and check if an `IOLoop` instance already exists.

```python
# The relative path of the corrected file: tornado/ioloop.py

# Corrected initialize function
def initialize(self, make_current=None):
    if make_current is None:
        if IOLoop.current(instance=False) is None:
            self.make_current()
    elif make_current:
        if IOLoop.current(instance=False) is not None:
            raise RuntimeError("current IOLoop already exists")
        self.make_current()
```
