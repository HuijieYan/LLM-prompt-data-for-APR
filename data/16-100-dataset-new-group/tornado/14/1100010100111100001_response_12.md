### Bug Explanation
The buggy function `initialize` in `tornado/ioloop.py` is designed to initialize the IOLoop, which is the event loop for handling asynchronous events in Tornado. The bug occurs when the `make_current` parameter is set to `True` and a current IOLoop already exists, resulting in a `RuntimeError` being raised.

### Bug Identification
The bug occurs in the following code block:
```python
elif make_current:
    if IOLoop.current(instance=False) is None:
        raise RuntimeError("current IOLoop already exists")
    self.make_current()
```
The `if` condition is incorrectly checking if the IOLoop does not exist and then raising a `RuntimeError` if it does not exist. This condition should be checking if the IOLoop already exists and then raising the error if it does.

### Bug Fix
To fix the bug, the condition in the `initialize` function needs to be corrected to check if the IOLoop already exists, and if it does, raise a `RuntimeError`, otherwise proceed with making the IOLoop current.

### Corrected Version
```python
def initialize(self, make_current=None):
    if make_current is None:
        if IOLoop.current(instance=False) is None:
            self.make_current()
    elif make_current:
        if IOLoop.current(instance=False) is not None:
            raise RuntimeError("current IOLoop already exists")
        self.make_current()
```
