### Analysis
The buggy function `initialize` is failing the test `test_force_current`. The error message indicates that the `RuntimeError` is being raised when trying to initialize the `IOLoop` with `make_current=True`. This suggests that the `initialize` function is not correctly handling the `make_current` parameter.

### Bug
The bug in the `initialize` function is that it doesn't correctly handle the case where `make_current` is `True`, and an `IOLoop` already exists.

### Fix
To fix the bug, the `initialize` function needs to check if an `IOLoop` already exists when `make_current` is `True`, and if so, raise a `RuntimeError`.

```python
# The relative path of the corrected file: tornado/ioloop.py

def initialize(self, make_current=None):
    if make_current is None:
        if IOLoop.current(instance=False) is None:
            self.make_current()
    elif make_current:
        if IOLoop.current(instance=False) is not None:
            raise RuntimeError("current IOLoop already exists")
        self.make_current()
```
