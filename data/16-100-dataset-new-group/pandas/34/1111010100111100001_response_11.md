The bug in the function `_get_time_bins` is causing an ambiguous time error due to the use of the `date_range` function. This error occurs because the function tries to create a date range with ambiguous time when converting the time from standard time to daylight saving time, leading to a `pytz.AmbiguousTimeError`. The bug seems to be related to the timezone conversion and the creation of the date range.

To fix the bug, we need to ensure that when creating the date range, the timezone information is preserved and handled properly.
