### Bug Explanation
The bug is causing an `AmbiguousTimeError` due to the handling of ambiguous timestamps on a long clock-change day. This occurs when the clock changes, making midnight a timestamp that is ambiguous. The issue is related to the code not handling this scenario correctly, causing the `groupby` call to raise the error.

### Bug Location
The buggy function `_get_time_bins` contains the logic to create bins and labels for grouping time series data. The error occurs due to how the function handles the time range on the long clock-change day.

### Bug Fix Strategy
To fix the bug, the logic in the `_get_time_bins` function needs to properly handle the ambiguous timestamp on the clock-change day. This could involve adjusting the time range or labels to avoid ambiguous timestamps.

        # Handle ambiguous time on clock-change day by using 'infer' and 'shift_forward'
