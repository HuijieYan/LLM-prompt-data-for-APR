The bug in the `pivot_table` function seems to stem from the recursive call to itself when `aggfunc` is a list. This recursive call is causing unexpected behavior and incorrect results.

To fix this bug, the recursive call to `pivot_table` for each `aggfunc` in the list needs to be replaced with a loop that iterates through each `aggfunc`. This will ensure that the function correctly applies each aggregation function to the data without the need for recursive calls.
