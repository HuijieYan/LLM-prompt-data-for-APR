## Bug Explanation
The `_try_convert_to_date` function is supposed to parse an array-like input into a date column. However, the bug causes it to return incorrect types of values, resulting in unexpected behavior when trying to convert boolean values to dates. This bug is described in the GitHub issue, where the user expected to receive a Pandas Series of boolean values but instead received a Pandas Series of datetime values or an exception.

The cause of the bug lies in the logic that checks for the data type and attempts to convert it to int64 before parsing it as a date. In the second case, the boolean values are incorrectly treated as datetime values, causing the unexpected behavior.

## Proposed Fix
The bug can be fixed by avoiding the unnecessary conversion to int64 for boolean values and updating the logic to handle boolean data correctly.

```python
def _try_convert_to_date(self, data):
    """
    Try to parse a ndarray-like input into a date column.

    Try to coerce object in epoch/iso formats and integer/float in epoch
    formats. Return a boolean if parsing was successful.
    """
    if not len(data):
        return data, False

    new_data = data
    if new_data.dtype == "object":
        try:
            if all(isinstance(x, (int, float)) for x in new_data):
                new_data = new_data.astype("int64")
            elif all(isinstance(x, bool) for x in new_data):
                new_data = new_data.astype("bool")
        except (TypeError, ValueError, OverflowError):
            pass

    if issubclass(new_data.dtype.type, np.number) or new_data.dtype == "bool":
        in_range = (isna(new_data._values) | (new_data > self.min_stamp) | (new_data._values == iNaT))
        if not in_range.all():
            return data, False

    date_units = (self.date_unit,) if self.date_unit else self._STAMP_UNITS
    for date_unit in date_units:
        try:
            new_data = to_datetime(new_data, errors="raise", unit=date_unit)
        except (ValueError, OverflowError):
            continue
        return new_data, True
    return data, False
```
