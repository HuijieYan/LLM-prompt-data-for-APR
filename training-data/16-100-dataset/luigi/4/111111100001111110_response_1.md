def copy(self, cursor, f):
    """
    Defines copying from s3 into redshift.

    If both key-based and role-based credentials are provided, role-based will be used.
    """
    logger.info("Inserting file: %s", f)
    colnames = ''
    if self.columns and len(self.columns) > 0:
        colnames = ",".join([x[0] for x in self.columns])
        colnames = '({})'.format(colnames)

    cursor.execute("""
     COPY {table} {colnames} from '{source}'
     CREDENTIALS '{creds}'
     {options}
     ;""".format(
        table=self.table,
        colnames=colnames,
        source=f,
        creds=self._credentials(),
        options=self.copy_options())
    )

In the fixed version of the function, I added a check for `self.columns` before attempting to get its length in the `copy` function. This will prevent the "object of type 'NoneType' has no len()" error when `self.columns` is `None`. This fix addresses the issue where Redshift COPY fails in luigi 2.7.1 when columns are not provided.
