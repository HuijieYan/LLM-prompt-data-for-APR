### Bug Cause
The issue arises from the `_convert_key` function in the `_AtIndexer` class inside the `pandas/core/indexing.py` file. When the key is a float and the index is a float index, the function incorrectly raises a ValueError due to not handling the float type correctly.

The failing test `test_lookups_datetimelike_values` in `pandas.tests.indexes.test_numeric.py` fails with a ValueError due to the incorrect handling of float index and float key in the `_convert_key` function.

The GitHub issue reflects that Series lookups are affected for the Float64Index case.

### Approaches for fixing the bug
To fix the bug, we need to update the `_convert_key` function to properly handle the float index and float keys. We should make sure that the function can correctly match index and key types for both integer and non-integer cases.

### The corrected code
```python
class _AtIndexer(_ScalarAccessIndexer):
    def _convert_key(self, key, is_setter: bool = False):
        """
        Require the keys to be the same type as the index. (so we don't
        fallback)
        """
        if is_setter:
            return list(key)

        for ax, i in zip(self.obj.axes, key):
            if is_numeric_dtype(ax):
                if not is_numeric_dtype(i):
                    raise ValueError(
                        "At based indexing on a numeric index "
                        "can only have numeric indexers"
                    )
            else:
                if is_numeric_dtype(i) and not is_numeric_dtype(ax):
                    raise ValueError(
                        "At based indexing on a non-numeric "
                        "index can only have non-numeric "
                        "indexers"
                    )
        return key
```

The corrected code updates the `_convert_key` function to handle both numeric and non-numeric indexes and indexers correctly. Now it should pass the failing test and resolve the issue reported in the GitHub thread.
